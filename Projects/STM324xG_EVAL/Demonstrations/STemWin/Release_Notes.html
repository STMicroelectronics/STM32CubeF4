Update History

V1.5.1 / 29-December-2017

Main Changes

  • Update stm32f4xx_hal_conf.h files to add support to HAL CAN Legacy Module.

V1.5.0 / 06-October-2017

Main Changes

  • Remove DATE and VERSION from the header of all the files.
  • Fix compilation issues with Linux
  • Remove support of TrueSTUDIO toolchain
  • Upgrade STemWin from 5.32 to 5.40

V1.4.0 / 17-February-2017

Main Changes

  • Update to be aligned with new HAL SD and BSP Drivers implementation

Known Limitations

  • TrueSTUDIO project is not provided
  • A generated noise may be heard when playing audio file, or when switching automatically to the next audio file

V1.3.6 / 04-November-2016

Main Changes

  • Update system_stm32f4xx.c file to be aligned with latest version of CMSIS STM32F4xx device implementation

V1.3.5 / 03-June-2016

Main Changes

  • Update demonstration to support STemWin 5.32 version

V1.3.4 / 06-May-2016

Main Changes

  • Add new define USE_SPI_CRC in stm32f4xx_hal_conf.h file for code cleanup when the CRC calculation is disabled

V1.3.3 / 29-January-2016

Main Changes

  • stm32f4xx_hal_conf.h:
    • Adjust the HSE Start-up Timeout value: 100ms instead of 5s
    • Add new define for the LSE start-up Timeout: LSE_STARTUP_TIMEOUT

V1.3.2 / 13-November-2015

Main Changes

  • system_stm32f4xx.c update to allow SDRAM & SRAM config at the same time
  • Update to use slash "/" instead of backslash "\" in the include path to avoid the Unix compilation errors

Known Limitations

  • TrueSTUDIO project is not provided

V1.3.1 / 14-August-2015

Main Changes

  • Update to support FatFS verion R0.11

Known Limitations

  • TrueSTUDIO project is not provided

V1.3.0 / 01-July-2015

Main Changes

  • Remove the video resizing in full screen mode
  • Update the video/audio pointer file management
  • Video Player module: update to clear the video file before launching the next video file in full screen mode
  • Update to disable ST kernel log, Process viewer and CPU usage with the video player module
  • Enhance GUI core message response by reducing the GUI task polling
  • Remove the full screen mode in video module module

Known Limitations

  • TrueSTUDIO project is not provided

V1.2.1 / 13-March-2015

Main Changes

  • Update following changes in latest version of HAL and Middlewares
  • All modules: update windows size to remove the white line appearing when exiting full screen mode and returning to images list menu
  • K_menu and system modules: 
    • fix flicker effect
    • change the demonstration date (2015 instead of 2014)
  • Miscellaneous enhancements

Known Limitations

  • TrueSTUDIO project is not provided

V1.2.0 / 26-December-2014

Main Changes

  • Update following changes in latest version of HAL and Middlewares
  • Update SystemClock_Config() to turn off the Flash prefetch if the code is running on STM32F407xx RevA devices, to comply with the errata sheet limitation
  • Remove vbus_sensing_enable from USBH_LL_Init()
  • Miscellaneous enhancements
  • Validation using latest toolchains version: EWARM v7.30

Known Limitations

  • TrueSTUDIO project is not provided
  • The following issues are detected when using EWARM V7.30, however they are not present when using EWARM V7.10, they will be fixed in next release:
    • Kernel Log doesnt display USB device, when detaching the USB key from the board
    • Kernel Log doesnt display Media files directory, after browsing any media file
    • Generated noise once unplugging a USB key from the board, once playing audio file, or when switching automatically to the next audio file

V1.1.0 / 26-June-2014

Main Changes

  • Update EWARM startup file to cope with compiler enhancement of the V7.x version
  • Update stm32f4xx_hal_conf.h to be in line with HAL V1.1.0: add new defines LSI_VALUE, LSE_VALUE and TICK_INT_PRIORITY
  • Update stm32f4xx_it.c : add the include of main.h before the stm32f4xx_it.h, in order to provide access to CMSIS device file via stm32f4xx_hal.h
  • Update to support the new STemWin version V5.24b
  • In AUDIO player module: customize the default SD_diskio driver and remove it from the project while a new file  k_SD_diskio.c is created in user folder, this implementation is done fix the issue on IO operation interruption by RTOS scheduler which could lead to overrun condition and as consequence the SDCard contents is not always displayed
  • Add new enhancement to support audio files with the extension ".WAV",  JPG, BMP and EMF (with upper case)
  • Miscellaneous source code comments update

Known Limitations

  • TrueSTUDIO project is not provided

V1.0.0 / 26-February-2014

Main Changes

  • First official release
